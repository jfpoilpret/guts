Guts-GUI is a GUI framework based on Google Guice dependency injection library and the Swing Application Framework (JSR-296).

Guts GUI framework API roadmap

{@link net.guts.gui.application} package contains the core classes to start a new application. As of version ${version}, there is only one possible main class to derive from in order to build your own application. For the time being, you can build an application with a main frame (and possibly secondary frames) and put whatever you want in this main frame (MDI or SDI). In the future, Guts-GUI will allow you to build a docking-based application with a single main frame containing dockable "views", allowing the end user to set the layout of views the way he wants. In any case, the framework provides exception handling facilities (as described in section ####).

In order to effectively use Guts-GUI framework, you'll have to know a little bit about Swing Application Framework upon which Guts-GUI is based. In particular you will need to know about:

  • Application lifecycle, in particular the following methods: {@link org.jdesktop.application.Application#startup()}, {@link org.jdesktop.application.Application#ready()}, {@link org.jdesktop.application.Application#shutdown()}. In addition, {@link org.jdesktop.application.Application$ExitListener}s are good to know.
  • Resources management (internationalization), in particular the way component properties are to be declared in properties files in order to be automatically injected, and how those files are searched by the framework.
  • Actions management: check the {@link org.jdesktop.application.Action @Action} annotation and its attributes.
The following links provide a good and quick introduction to Swing Application Framework:
  • An Introduction to the Swing Application Framework API (JSR-296)
  • Using the Swing Application Framework (JSR 296): very good introduction article by John O'Conner
  • Dean Iverson's tutorial
Moreover, you may take a look at this Jazoon'08 talk for deeper coverage of Swing Application Framework based on real-life usage (part of Guts-GUI framework was developed for this presentation).

Additionally, you need to have some basic knowledge about Guice Depedency Injection library (from Google) which is at the core of Guts-GUI framework. In particular, you need to understand the {@code @Inject} annotation and the notion of {@code Module}s. The following links provide good startup material on Guice:

  • Guice presentation by one Guice committer
  • Guice user's guide

Besides, Guts-GUI provides several facilities as "services" (defined and accessed by their interface) that will help you:

  • Manage your {@code @Action} methods easily
  • Easily create menus and popups linked to {@code @Action} methods
  • Easily create and display internationalized dialogs with a consistent look & feel
  • Create and show internationalized messages boxes
Plus, a few utility classes gathered in package {@link net.guts.gui.util}.
