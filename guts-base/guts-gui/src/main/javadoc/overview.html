Guts-GUI is a GUI framework based on Google Guice dependency injection library and inspired by the Swing Application Framework (JSR-296).

Guts GUI framework API roadmap

{@code net.guts.gui.application} package contains the core classes to start a new application.

In order to effectively use Guts-GUI framework, you'll need to have some basic knowledge about Guice Dependency Injection library (from Google) which is at the core of Guts-GUI framework. In particular, you need to understand the {@code @Inject} annotation and the notion of {@code Module}s. The following links provide good startup material on Guice:

  • Guice presentation by one Guice committer
  • Guice user's guide

Here is the API description for what the Guts-GUI framework provides:

  • Lifecycle management including exception handling and shutdown management
  • Lifecycle support for useful abstract {@linkplain net.guts.gui.application.AppLifecycleStarter}s.
  • Window management
  • Action Management including resource injection, and standard actions support such as cut, copy, paste
  • Menu factory support in relation to actions management above
  • Automatic Resource Injection of Swing components for internationalization purposes
  • Special support for launching long tasks without blocking the Swing Event Dispatch Thread
  • Automatic Session Storage allowing for restoring, across launches, the application real estate as set by the user (e.g. window position and size, table column headers width...)
  • Support for dialogs creation
  • Support for internationalized Message Boxes
  • Special "template" system that allows consistent decoration of window content
  • Since Resource Injection and Session Storage depend on all components' names (as set by {@code Component.setName()}), support for automatically naming components is provided, in order to remove the boilerplate code for naming
  • A few general utility classes are also provided
Guts-GUI is general and can support any kind of application:
  • Single Frame applications
  • Multiple Frames applications
  • MDI-based applications
  • Docking-based applications
  • Dialog-based applications (e.g. installers)
